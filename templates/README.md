# Templates

Esta pasta reúne os modelos oficiais para manter os recursos consistentes e verificáveis. Use o template correspondente sempre que criar um agente, prompt, workflow, skill ou documento de contexto.

Modelos disponíveis:

- [agente](agent-template.md);
- [prompt](prompt-template.md);
- [workflow](workflow-template.md);
- [skill](skill-template.md), para skills próprias com `SKILL.md` e arquivos auxiliares;
- [contexto de projeto](project-context-template.md).
- [registro de recurso de terceiros](../third-party/resource-record-template.md), para documentar procedência, licença, versão, revisão e decisão de curadoria de materiais externos.

Preserve o front matter YAML, adapte somente os campos pertinentes e use nomes em minúsculas com hífens. Nunca preencha exemplos públicos com dados pessoais, profissionais ou confidenciais. Templates preenchidos com contexto particular pertencem ao repositório privado. Atualize o [catálogo](../CATALOG.md) no mesmo commit quando um template passar a representar ou alterar recursos catalogados.
