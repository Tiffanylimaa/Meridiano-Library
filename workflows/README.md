# Workflows

Esta pasta documenta processos reutilizáveis com várias etapas. Crie um workflow quando uma tarefa tiver sequência definida, dependências, decisões, aprovações humanas ou validações finais.

Cada workflow deve informar resultado esperado, gatilho, pré-requisitos, entradas, etapas, pontos de aprovação humana, tratamento de erros, validação e saídas. Categorias recomendadas incluem conteúdo, pesquisa, qualidade e operações, somente quando houver materiais concretos.

Não incorpore processos internos ou dados confidenciais. Use o [template de workflow](../templates/workflow-template.md) e atualize o [catálogo](../CATALOG.md) no mesmo commit em qualquer criação, movimentação, descontinuação ou arquivamento.
